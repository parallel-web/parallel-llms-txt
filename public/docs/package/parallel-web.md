<div class="markdown-heading"><h3 class="heading-element">Retries</h3><a id="user-content-retries" class="anchor" aria-label="Permalink: Retries" href="#retries"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>Certain errors will be automatically retried 2 times by default, with a short exponential backoff.
Connection errors (for example, due to a network connectivity problem), 408 Request Timeout, 409 Conflict,
429 Rate Limit, and &gt;=500 Internal errors will all be retried by default.</p>
<p>You can use the <code>maxRetries</code> option to configure or disable this:</p>

<div class="highlight highlight-source-js"><pre><span class="pl-c">// Configure the default for all requests:</span>
<span class="pl-k">const</span> <span class="pl-s1">client</span> <span class="pl-c1">=</span> <span class="pl-k">new</span> <span class="pl-v">Parallel</span><span class="pl-kos">(</span><span class="pl-kos">{</span>
  <span class="pl-c1">maxRetries</span>: <span class="pl-c1">0</span><span class="pl-kos">,</span> <span class="pl-c">// default is 2</span>
<span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">;</span>

<span class="pl-c">// Or, configure per-request:</span>
<span class="pl-k">await</span> <span class="pl-s1">client</span><span class="pl-kos">.</span><span class="pl-c1">taskRun</span><span class="pl-kos">.</span><span class="pl-en">create</span><span class="pl-kos">(</span><span class="pl-kos">{</span> <span class="pl-c1">input</span>: <span class="pl-s">'What was the GDP of France in 2023?'</span><span class="pl-kos">,</span> <span class="pl-c1">processor</span>: <span class="pl-s">'base'</span> <span class="pl-kos">}</span><span class="pl-kos">,</span> <span class="pl-kos">{</span>
  <span class="pl-c1">maxRetries</span>: <span class="pl-c1">5</span><span class="pl-kos">,</span>
<span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">;</span></pre></div>
<div class="markdown-heading"><h3 class="heading-element">Timeouts</h3><a id="user-content-timeouts" class="anchor" aria-label="Permalink: Timeouts" href="#timeouts"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>Requests time out after 1 minute by default. You can configure this with a <code>timeout</code> option:</p>

<div class="highlight highlight-source-ts"><pre><span class="pl-c">// Configure the default for all requests:</span>
<span class="pl-k">const</span> <span class="pl-s1">client</span> <span class="pl-c1">=</span> <span class="pl-k">new</span> <span class="pl-v">Parallel</span><span class="pl-kos">(</span><span class="pl-kos">{</span>
  <span class="pl-c1">timeout</span>: <span class="pl-c1">20</span> <span class="pl-c1">*</span> <span class="pl-c1">1000</span><span class="pl-kos">,</span> <span class="pl-c">// 20 seconds (default is 1 minute)</span>
<span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">;</span>

<span class="pl-c">// Override per-request:</span>
<span class="pl-k">await</span> <span class="pl-s1">client</span><span class="pl-kos">.</span><span class="pl-c1">taskRun</span><span class="pl-kos">.</span><span class="pl-en">create</span><span class="pl-kos">(</span><span class="pl-kos">{</span> <span class="pl-c1">input</span>: <span class="pl-s">'What was the GDP of France in 2023?'</span><span class="pl-kos">,</span> <span class="pl-c1">processor</span>: <span class="pl-s">'base'</span> <span class="pl-kos">}</span><span class="pl-kos">,</span> <span class="pl-kos">{</span>
  <span class="pl-c1">timeout</span>: <span class="pl-c1">5</span> <span class="pl-c1">*</span> <span class="pl-c1">1000</span><span class="pl-kos">,</span>
<span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">;</span></pre></div>
<p>On timeout, an <code>APIConnectionTimeoutError</code> is thrown.</p>
<p>Note that requests which time out will be <a href="#retries">retried twice by default</a>.</p>

<div class="markdown-heading"><h3 class="heading-element">Accessing raw Response data (e.g., headers)</h3><a id="user-content-accessing-raw-response-data-eg-headers" class="anchor" aria-label="Permalink: Accessing raw Response data (e.g., headers)" href="#accessing-raw-response-data-eg-headers"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>The "raw" <code>Response</code> returned by <code>fetch()</code> can be accessed through the <code>.asResponse()</code> method on the <code>APIPromise</code> type that all methods return.

  <span class="pl-kos">.</span><span class="pl-en">asResponse</span><span class="pl-kos">(</span><span class="pl-kos">)</span><span class="pl-kos">;</span>

<div class="markdown-heading"><h3 class="heading-element">Logging</h3><a id="user-content-logging" class="anchor" aria-label="Permalink: Logging" href="#logging"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>

<div class="markdown-heading"><h4 class="heading-element">Log levels</h4><a id="user-content-log-levels" class="anchor" aria-label="Permalink: Log levels" href="#log-levels"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>The log level can be configured in two ways:</p>
<ol>
<li>Via the <code>PARALLEL_LOG</code> environment variable</li>
<li>Using the <code>logLevel</code> client option (overrides the environment variable if set)</li>
</ol>
<div class="highlight highlight-source-ts"><pre><span class="pl-k">import</span> <span class="pl-v">Parallel</span> <span class="pl-k">from</span> <span class="pl-s">'parallel-web'</span><span class="pl-kos">;</span>

<span class="pl-k">const</span> <span class="pl-s1">client</span> <span class="pl-c1">=</span> <span class="pl-k">new</span> <span class="pl-v">Parallel</span><span class="pl-kos">(</span><span class="pl-kos">{</span>
  <span class="pl-c1">logLevel</span>: <span class="pl-s">'debug'</span><span class="pl-kos">,</span> <span class="pl-c">// Show all log messages</span>
<span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">;</span></pre></div>
<p>Available log levels, from most to least verbose:</p>
<ul>
<li>
<code>'debug'</code> - Show debug messages, info, warnings, and errors</li>
<li>
<code>'info'</code> - Show info messages, warnings, and errors</li>
<li>
<code>'warn'</code> - Show warnings and errors (default)</li>
<li>
<code>'error'</code> - Show only errors</li>
<li>
<code>'off'</code> - Disable all logging</li>
</ul>
<p>At the <code>'debug'</code> level, all HTTP requests and responses are logged, including headers and bodies.
Some authentication-related headers are redacted, but sensitive data in request and response bodies
may still be visible.</p>
<div class="markdown-heading"><h4 class="heading-element">Custom logger</h4><a id="user-content-custom-logger" class="anchor" aria-label="Permalink: Custom logger" href="#custom-logger"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>By default, this library logs to <code>globalThis.console</code>. You can also provide a custom logger.
Most logging libraries are supported, including <a href="https://www.npmjs.com/package/pino" rel="nofollow">pino</a>, <a href="https://www.npmjs.com/package/winston" rel="nofollow">winston</a>, <a href="https://www.npmjs.com/package/bunyan" rel="nofollow">bunyan</a>, <a href="https://www.npmjs.com/package/consola" rel="nofollow">consola</a>, <a href="https://www.npmjs.com/package/signale" rel="nofollow">signale</a>, and <a href="https://jsr.io/@std/log" rel="nofollow">@std/log</a>. If your logger doesn't work, please open an issue.</p>
<p>When providing a custom logger, the <code>logLevel</code> option still controls which messages are emitted, messages
below the configured level will not be sent to your logger.</p>
<div class="highlight highlight-source-ts"><pre><span class="pl-k">import</span> <span class="pl-v">Parallel</span> <span class="pl-k">from</span> <span class="pl-s">'parallel-web'</span><span class="pl-kos">;</span>
<span class="pl-k">import</span> <span class="pl-s1">pino</span> <span class="pl-k">from</span> <span class="pl-s">'pino'</span><span class="pl-kos">;</span>

<span class="pl-k">const</span> <span class="pl-s1">logger</span> <span class="pl-c1">=</span> <span class="pl-en">pino</span><span class="pl-kos">(</span><span class="pl-kos">)</span><span class="pl-kos">;</span>

<span class="pl-k">const</span> <span class="pl-s1">client</span> <span class="pl-c1">=</span> <span class="pl-k">new</span> <span class="pl-v">Parallel</span><span class="pl-kos">(</span><span class="pl-kos">{</span>
  <span class="pl-c1">logger</span>: <span class="pl-s1">logger</span><span class="pl-kos">.</span><span class="pl-en">child</span><span class="pl-kos">(</span><span class="pl-kos">{</span> <span class="pl-c1">name</span>: <span class="pl-s">'Parallel'</span> <span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">,</span>
  <span class="pl-c1">logLevel</span>: <span class="pl-s">'debug'</span><span class="pl-kos">,</span> <span class="pl-c">// Send all messages to pino, allowing it to filter</span>
<span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">;</span></pre></div>
<div class="markdown-heading"><h3 class="heading-element">Making custom/undocumented requests</h3><a id="user-content-making-customundocumented-requests" class="anchor" aria-label="Permalink: Making custom/undocumented requests" href="#making-customundocumented-requests"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>This library is typed for convenient access to the documented API. If you need to access undocumented
endpoints, params, or response properties, the library can still be used.</p>
<div class="markdown-heading"><h4 class="heading-element">Undocumented endpoints</h4><a id="user-content-undocumented-endpoints" class="anchor" aria-label="Permalink: Undocumented endpoints" href="#undocumented-endpoints"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>To make requests to undocumented endpoints, you can use <code>client.get</code>, <code>client.post</code>, and other HTTP verbs.
Options on the client, such as retries, will be respected when making these requests.</p>
<div class="highlight highlight-source-ts"><pre><span class="pl-k">await</span> <span class="pl-s1">client</span><span class="pl-kos">.</span><span class="pl-en">post</span><span class="pl-kos">(</span><span class="pl-s">'/some/path'</span><span class="pl-kos">,</span> <span class="pl-kos">{</span>
  <span class="pl-c1">body</span>: <span class="pl-kos">{</span> <span class="pl-c1">some_prop</span>: <span class="pl-s">'foo'</span> <span class="pl-kos">}</span><span class="pl-kos">,</span>
  <span class="pl-c1">query</span>: <span class="pl-kos">{</span> <span class="pl-c1">some_query_arg</span>: <span class="pl-s">'bar'</span> <span class="pl-kos">}</span><span class="pl-kos">,</span>
<span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">;</span></pre></div>
<div class="markdown-heading"><h4 class="heading-element">Undocumented request params</h4><a id="user-content-undocumented-request-params" class="anchor" aria-label="Permalink: Undocumented request params" href="#undocumented-request-params"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>To make requests using undocumented parameters, you may use <code>// @ts-expect-error</code> on the undocumented
parameter. This library doesn't validate at runtime that the request matches the type, so any extra values you
send will be sent as-is.</p>
<div class="highlight highlight-source-ts"><pre><span class="pl-s1">client</span><span class="pl-kos">.</span><span class="pl-c1">taskRun</span><span class="pl-kos">.</span><span class="pl-en">create</span><span class="pl-kos">(</span><span class="pl-kos">{</span>
  <span class="pl-c">// ...</span>
  <span class="pl-c">// <span class="pl-k">@ts</span>-expect-error baz is not yet public</span>
  <span class="pl-c1">baz</span>: <span class="pl-s">'undocumented option'</span><span class="pl-kos">,</span>
<span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">;</span></pre></div>
<p>For requests with the <code>GET</code> verb, any extra params will be in the query, all other requests will send the
extra param in the body.</p>
<p>If you want to explicitly send an extra argument, you can do so with the <code>query</code>, <code>body</code>, and <code>headers</code> request
options.</p>
<div class="markdown-heading"><h4 class="heading-element">Undocumented response properties</h4><a id="user-content-undocumented-response-properties" class="anchor" aria-label="Permalink: Undocumented response properties" href="#undocumented-response-properties"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>To access undocumented response properties, you may access the response object with <code>// @ts-expect-error</code> on
the response object, or cast the response object to the requisite type. Like the request params, we do not
validate or strip extra properties from the response from the API.</p>
<div class="markdown-heading"><h3 class="heading-element">Customizing the fetch client</h3><a id="user-content-customizing-the-fetch-client" class="anchor" aria-label="Permalink: Customizing the fetch client" href="#customizing-the-fetch-client"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>By default, this library expects a global <code>fetch</code> function is defined.</p>
<p>If you want to use a different <code>fetch</code> function, you can either polyfill the global:</p>
<div class="highlight highlight-source-ts"><pre><span class="pl-k">import</span> <span class="pl-s1">fetch</span> <span class="pl-k">from</span> <span class="pl-s">'my-fetch'</span><span class="pl-kos">;</span>

<span class="pl-s1">globalThis</span><span class="pl-kos">.</span><span class="pl-c1">fetch</span> <span class="pl-c1">=</span> <span class="pl-s1">fetch</span><span class="pl-kos">;</span></pre></div>
<p>Or pass it to the client:</p>
<div class="highlight highlight-source-ts"><pre><span class="pl-k">import</span> <span class="pl-v">Parallel</span> <span class="pl-k">from</span> <span class="pl-s">'parallel-web'</span><span class="pl-kos">;</span>
<span class="pl-k">import</span> <span class="pl-s1">fetch</span> <span class="pl-k">from</span> <span class="pl-s">'my-fetch'</span><span class="pl-kos">;</span>

<span class="pl-k">const</span> <span class="pl-s1">client</span> <span class="pl-c1">=</span> <span class="pl-k">new</span> <span class="pl-v">Parallel</span><span class="pl-kos">(</span><span class="pl-kos">{</span> fetch <span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">;</span></pre></div>
<div class="markdown-heading"><h3 class="heading-element">Fetch options</h3><a id="user-content-fetch-options" class="anchor" aria-label="Permalink: Fetch options" href="#fetch-options"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>If you want to set custom <code>fetch</code> options without overriding the <code>fetch</code> function, you can provide a <code>fetchOptions</code> object when instantiating the client or making a request. (Request-specific options override client options.)</p>
<div class="highlight highlight-source-ts"><pre><span class="pl-k">import</span> <span class="pl-v">Parallel</span> <span class="pl-k">from</span> <span class="pl-s">'parallel-web'</span><span class="pl-kos">;</span>

<span class="pl-k">const</span> <span class="pl-s1">client</span> <span class="pl-c1">=</span> <span class="pl-k">new</span> <span class="pl-v">Parallel</span><span class="pl-kos">(</span><span class="pl-kos">{</span>
  <span class="pl-c1">fetchOptions</span>: <span class="pl-kos">{</span>
    <span class="pl-c">// `RequestInit` options</span>
  <span class="pl-kos">}</span><span class="pl-kos">,</span>
<span class="pl-kos">}</span><span class="pl-kos">)</span><span class="pl-kos">;</span></pre></div>

<div class="markdown-heading"><h2 class="heading-element">Semantic versioning</h2><a id="user-content-semantic-versioning" class="anchor" aria-label="Permalink: Semantic versioning" href="#semantic-versioning"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>This package generally follows <a href="https://semver.org/spec/v2.0.0.html" rel="nofollow">SemVer</a> conventions, though certain backwards-incompatible changes may be released as minor versions:</p>
<ol>
<li>Changes that only affect static types, without breaking runtime behavior.</li>
<li>Changes to library internals which are technically public but not intended or documented for external use. <em>(Please open a GitHub issue to let us know if you are relying on such internals.)</em>
</li>
<li>Changes that we do not expect to impact the vast majority of users in practice.</li>
</ol>
<p>We take backwards-compatibility seriously and work hard to ensure you can rely on a smooth upgrade experience.</p>
<p>We are keen for your feedback; please open an <a href="https://www.github.com/parallel-web/parallel-sdk-typescript/issues">issue</a> with questions, bugs, or suggestions.</p>
<div class="markdown-heading"><h2 class="heading-element">Requirements</h2><a id="user-content-requirements" class="anchor" aria-label="Permalink: Requirements" href="#requirements"><span aria-hidden="true" class="octicon octicon-link"></span></a></div>
<p>TypeScript &gt;= 4.9 is supported.</p>
<p>The following runtimes are supported:</p>
<ul>
<li>Web browsers (Up-to-date Chrome, Firefox, Safari, Edge, and more)</li>
<li>Node.js 20 LTS or later (<a href="https://endoflife.date/nodejs" rel="nofollow">non-EOL</a>) versions.</li>
<li>Deno v1.28.0 or higher.</li>
<li>Bun 1.0 or later.</li>
<li>Cloudflare Workers.</li>
<li>Vercel Edge Runtime.</li>
<li>Jest 28 or greater with the <code>"node"</code> environment (<code>"jsdom"</code> is not supported at this time).</li>
<li>Nitro v2.6 or greater.</li>
</ul>
<p>Note that React Native is not supported at this time.</p>
<p>If you are interested in other runtime environments, please open or upvote an issue on GitHub.</p>
